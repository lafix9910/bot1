# Telegram Bot для записи на маникюр

Полностью рабочий Telegram-бот для салона красоты с функционалом записи на услуги.

## Возможности

### Клиент:
- 📅 Запись на услуги (маникюр, педикюр, дизайн)
- 👤 Выбор мастера
- 📆 Выбор даты и времени
- 📋 Просмотр своих записей
- 🔄 Перенос записи
- ❌ Отмена записи

### Админ:
- 📋 Просмотр всех записей
- ✅ Подтверждение/отмена записей
- 👥 Добавление мастеров
- ⏰ Настройка рабочих часов

## Установка и запуск

### 1. Создание бота через BotFather

1. Откройте Telegram и найдите @BotFather
2. Отправьте команду `/newbot`
3. Введите имя бота (например, "Nail Salon Bot")
4. Введите username бота (например, `nail_salon_bot`)
5. Скопируйте токен бота (формат: `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`)

### 2. Настройка переменных окружения

Создайте файл `.env` в корне проекта:

```env
BOT_TOKEN=ваш_токен_бота
ADMIN_IDS=123456789,987654321
ADMIN_USERNAMES=username1,username2

# Опционально (по умолчанию):
WORKING_HOURS_START=9
WORKING_HOURS_END=20
SLOT_DURATION_MINUTES=60
DAYS_IN_ADVANCE=14
```

- `BOT_TOKEN` - токен от BotFather
- `ADMIN_IDS` - ID администраторов через запятую (узнать можно через @userinfobot)
- `ADMIN_USERNAMES` - username админов без @ через запятую

### 3. Установка зависимостей

```bash
cd nail-bot
pip install -r requirements.txt
```

### 4. Запуск бота

```bash
python main.py
```

или

```bash
python main.py
```

## Команды

- `/start` - Главное меню
- `/mybookings` - Мои записи
- `/admin` - Админ-панель (только для админов)

## База данных

Используется SQLite. База создаётся автоматически при первом запуске (`nail_bot.db`).

При первом запуске автоматически добавляются:
- Услуги: Маникюр, Педикюр, Дизайн, Маникюр+Педикюр, Снятие покрытия
- Мастера: Анна, Мария, Елена

## Требования

- Python 3.11+
- aiogram 3.4.1+
- SQLAlchemy 2.0+
- python-dotenv 1.0+

## Логика работы

1. Пользователь выбирает "Записаться"
2. Выбирает услугу из списка
3. Выбирает мастера
4. Выбирает дату из календаря (14 дней вперёд)
5. Выбирает свободное время
6. Подтверждает запись
7. Админ получает уведомление о новой записи
8. Админ подтверждает или отклоняет запись
